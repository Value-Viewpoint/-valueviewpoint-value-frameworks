# Contributing Guidelines

Thank you for your interest in contributing to the
**ValueViewpoint Value Engineering Frameworks** repository.

This repository is intended to share **open, finance-grade value engineering
methodology**, not product code or customer-specific materials.

---

## What Contributions Are Welcome

We welcome contributions that improve the **clarity, rigor, and usability**
of value engineering frameworks, including:

- Refinements to existing frameworks or schemas
- Additional worked examples using synthetic data
- Clarifications to assumptions, calculation logic, or terminology
- Suggestions for new value driver categories or patterns
- Improvements to documentation structure and readability

All contributions should remain **tool-agnostic** and **vendor-neutral**.

---

## What Is NOT Accepted

To protect trust and confidentiality, please do **not** submit:

- Customer-specific data or benchmarks
- Proprietary or confidential information
- Vendor marketing claims or performance guarantees
- Product source code or implementation logic
- Unverifiable or non-finance-traceable assumptions

Submissions that include such content will not be accepted.

---

## Contribution Process

1. Open an **Issue** to describe the proposed change or addition
2. Clearly explain the rationale and expected impact
3. Submit changes via a **Pull Request**
4. Ensure all examples use synthetic or anonymized data
5. Provide clear documentation and reasoning for any calculations

All contributions are reviewed for:
- Economic rigor
- Finance traceability
- Consistency with existing schemas
- Neutral and professional tone

---

## Review and Governance

This repository is curated to reflect **enterprise and finance standards**.
Maintainers may request revisions or clarifications before accepting changes.

Acceptance of a contribution does not imply endorsement of
any specific vendor, product, or implementation approach.

---

---

## Questions or Suggestions

If you are unsure whether a contribution is appropriate,
please open an Issue to discuss it before submitting changes.

Thank you for helping advance disciplined, transparent
value engineering practices.
